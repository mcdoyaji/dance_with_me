처리 중

필요한 후속조치

안 끝났음

자초지정

초급에게 일을 나눠주려고 하는데,  모든화면 + 기본 API 통신 방법은 제공해야 함. 핵심 기능인 채팅을 하려는데 2주 중 1주를 서버 오류로 삽질만 함.

처리

인증 + API 요청

- vue-api-query + vuex store action 결합
- 로그인/아웃 : 위 코드 연동
- 새로 고쳐도 인증 유지 : vue-persistedstate
- 인증 성공 시 헤더 설정

12/08 vue-api-query 공식문서가 좀 부실한 거 같다. find(), params() 가 필요하단 걸 결국은 알아냈다. 초급개발자에게 typescript 인터페이스, 타입 만들고 dto 만들어서 ... 여튼. 액션 -> vue-api-query->뮤테이션-> 스테이트 접근 값 사용까지 한 사이클 함께 돌아봤다. quasar 문서, 이것도 생각보다 보기가 어렵다.

파일 업로드를 vue-api-query 로 어떻게 해야 할 지를 모르겠네. 초급이 할 수 있을까?

https://github.com/robsontenorio/vue-api-query/issues/83

https://robsontenorio.github.io/vue-api-query/#configuring-formdata

채팅 + 웹소켓

- socket.io 연결
- vue-socket.io 에다가 vuex action 연동 
- 이벤트 수신 시 action 작동 -> 뮤테이션 -> 화면 반영
- 처음에 API 조회 -> 기본 데이터 생성 -> 이벤트 수신시 기존 것에 더하기
- 뮤테이션은 언제 만드냐. base -> impl 요리요리 계속 타입 만드는 것도 일이다.
 - api return 타입 만들고, 그걸 실제 저장하고 사용하는 class 도 만들고...

추가화면 뽑아내기

- 팝업 다이얼로그 하나에 왜 그리 많은 API 와 기능이 들어갈까.
- API 호출하는 부분 빼고, UI, 기능, API 호출 뒤 응답처리 등등 하니 하루종일 두 개 쳤다.
- 이것 때문에 나도 할 일이 있는데 밀린다. 조직에서는 내가 하는 일이 더 우선순위인데.
- 언제 몰아서 모든 화면을 다 뽑아주는 게 좋겠다. 병행은 못하겠어.
